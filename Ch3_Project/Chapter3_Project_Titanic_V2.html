Project "Titanic"¶

1. Approach and Dataset¶

The data set was obtained from the Udacity project homepage. It contains demographics and passenger information from 891 of the 2224 passengers and crew on board the Titanic. A brief description of the circumstances of this catastrophe can be viewed on the Kaggle website.

The description of the columns, i.e. a short data dictionary, is also available from the Kaggle website. The following table shows its contents:

Variable Definition Key
survived Survival 0 = No, 1 = Yes
pclass Ticket class 1 = 1st, 2 = 2nd, 3 = 3rd.
sex Sex male, female
Age Age in years
sibsp No. of siblings / spouses aboard the Titanic
parch No. of parents / children aboard the Titanic
ticket Ticket number
fare Passenger fare
cabin Cabin number
embarked Port of Embarkation C = Cherbourg, Q = Queenstown, S = Southampton

fare: Currency not given in the data dictionary, probably Pound Sterling (£)

pclass: A proxy for socio-economic status (SES) (1st = Upper, 2nd = Middle, 3rd = Lower)

age: Age is fractional if less than 1. If the age is estimated, is it in the form of xx.5

sibsp: The dataset defines family relations in this way.

  • Sibling = brother, sister, stepbrother, stepsister
  • Spouse = husband, wife (mistresses and fiancés were ignored)

parch: The dataset defines family relations in this way

  • Parent = mother, father
  • Child = daughter, son, stepdaughter, stepson
  • Some children travelled only with a nanny, therefore parch=0 for them.

3. Possible Questions¶

I would like to investigate the following questions:

  1. What are the determinants of survival? How are socio-economic status (Pclass), sex, age and family relationships (Parch) related to survival (Survived)?
  2. How is age and ticket fare related, i.e. are older passengers on average more wealthy?

The motivation behind these questions is to better understand the situation aboard the Titanic during the catastrophe and the types of people involved as passengers. Unlike any current, business-related data set, this historic data set cannot be analyzed in a way that enables us to derive actionable insights for similar situations in the future (i.e. derive a model with predictive power), as almost all naval, especially technical circumstances have changed over the last decades, and security standards have been raised.

4.2 Data cleaning¶

The data cleaning steps for the Titanic data set are based on source code from this website. Only minor actions are necessary:

  • The columns "Age", "Cabin" and "Embarked" contain missing values. According to this article, there are several options for replacing missing values.
    • The missing values in Cabin and Embarked are replaced by the constant string "Unknown".
    • "Age" as a column will not be deleted as only a rather low percentage of values are missing. I would assume that the missing values are missing completely at random (MCAR), i.e. they do not introduce bias. The correct treatment should depend on the further processing of this column. If I am only interested in a one-dimensional analysis or visualization, I will ignore the missing values, as the remaining values still deliver a good approximation of the overall age distribution. If I am interested in a two-dimensional analysis, and the second dimension besides "Age" has no missing values, I will still ignore the missing Age values. This approach has been chosen for this project. (If I needed to train a multivariate model, I would have to replace the missing Age values with either the median or the mean, because many multivariate models will ignore data records with missing values during training (or they automatically impute values)).
  • a new column Section is created, based on the letter codes in Cabin. The location of the sections can be found on the Titanic deckplan. The utility of this column remains to be seen because of the high number of missing values.
  • another new column Family_Size is created as the sum of the SibSp and the Parch fields, i.e. the sum of all relatives on board.
  • another new column AgeType is created, which describes typical age ranges.
  • adapt the data types of the columns PassengerId, Pclass. Those were loaded as int64 data types, but they are really nominal data types with integer factor labels (see Level of measurement). It makes no sense to use these columns for numerical calculations. They have an inherent order, so the correct level or measurement would be ordinal, but the astype() function would not accept the "ordered=True" parameter (although it is documented for series).
  • the column "Survived" could either be modeled as a categorical or a numerical variable. Treating survived as a numerical variable makes it easier to aggregate its contents (e.g. summing up the survivors). Therefore it remains numerical.

import string

def substrings_in_string(big_string, substrings):
    u"""looks for the first occcurence of one of the substrings in big_string"""
    for substring in substrings:
        if string.find(big_string, substring) != -1:
            return substring
    print big_string
    return np.nan

#Special case for cabins as NaN is nonsensical
titanic_df.Cabin = titanic_df.Cabin.fillna('Unknown')
titanic_df.Embarked = titanic_df.Embarked.fillna('Unknown')

#Turning cabin number into Sections
cabin_list = ['A', 'B', 'C', 'D', 'E', 'F', 'T', 'G', 'Unknown']
titanic_df['Section']=titanic_df['Cabin'].map(lambda x: substrings_in_string(x, cabin_list))
        
#Creating new family_size column
titanic_df['Family_Size']=titanic_df['SibSp']+titanic_df['Parch']

titanic_df['PassengerId'] = titanic_df.PassengerId.astype('category')
titanic_df['Pclass'] = titanic_df['Pclass'].astype('object')

# Binning of column Age
titanic_df['AgeType'] = pd.cut(titanic_df['Age'], bins = [0, 15, 60, 100], labels = ['Kids', 'Adults', 'Seniors'])

5. Exploration¶

5.1 Question 1¶

In the following paragraphs, I am going to describe my approach for answering question 1, which focuses on various factors determining the survival of the passengers. As the project guidelines specifically mention not to use statistics or machine learning to complete this project, I will resort to more fundamental techniques.

As the column Survived is a numeric column, one could think of using (linear) correlation (Pearson's R) between numeric columns, but this approach is largely not applicable for this data set, as we would need a ratio scale for the two correlated columns. The column Survived is only a binary (nominal) flag and hence not suitable for correlations.

Therefore, this project uses group comparisons between the Survived column and one or two other columns, e.g. (Survived, Section), (Survived, Sex), and computes statistical measures on the two groups (survivors, non-survivors). The results will also be well suited for later visualization.

Using this approach delivers descriptive statistics, but it won't allow the research to qualify the statistical significance of the perceived differences. Moreover, the project operates on a sample data set, so the findings will be tentative.

6. Conclusion¶

6.1 Limitations of the analysis¶

  • a high number of null values in the columns Age and Cabin makes it hard to draw conclusions from these columns that match those of other columns.
  • this sample of small size makes it hard to generalize answers for the population; moreover, it is not clear, how the elements in the sample were drawn from the population. The sampling process could have led to a biased sample.
  • Using group comparisons delivers descriptive statistics, but it won't allow the researcher to qualify the statistical significance of the perceived differences. A statistical test would be required for that.

6.3 Further Research¶

This paragraph briefly explains what other analyses may be performed on this data set based on what was presented in my report.

I would like to pick up on a topic suggestion of one of the reviewers: fare is a particularly tough variable to deal with in this data set. It is unknown whether it corresponds to the individual value paid by each passenger, or to the value paid for the ticket as a whole - which might include people traveling together (such as family members).

I would approach this additional analysis by looking at the (linear) regression between the Fare column and the columns SibSp,Parch and Family_Size. The regression delivers the coefficients and the intercept, which could give a hint to the tariff system, from which the fare was calculated.
